# Community Guidelines

This project welcomes contributors focused on reliable, scalable GIS infrastructure.

## How To Participate

- Open issues for bugs, data-quality concerns, and enhancement requests.
- Keep changes scoped and clearly documented.
- Include context for any map or routing behavior changes.

## Development Norms

- Prefer reproducible infrastructure and deployment steps.
- Separate environment-specific settings from shared templates.
- Prioritize performance and correctness for geospatial workloads.

## Conduct

- Be respectful and constructive in all discussions.
- Review feedback objectively and iterate quickly.
